## 第一部分：了解 OAuth 流程

### 什么是 OAuth？

OAuth 是一种安全协议，允许用户授权第三方应用访问其信息，而无需分享密码。

### Second Me OAuth 流程

```
┌─────────────────┐     1. 点击登录      ┌─────────────────┐
│  MedRoundTable  │ ───────────────────> │   Second Me     │
│   (你的应用)     │                      │  (身份提供商)    │
└─────────────────┘                      └─────────────────┘
         │                                        │
         │  3. 授权码 + 用户同意                    │
         │ <───────────────────────────────────── │
         │                                        │
         │  4. 用授权码换取 Token                   │
         │ ─────────────────────────────────────> │
         │                                        │
         │  5. 返回 Access Token                    │
         │ <───────────────────────────────────── │
         │                                        │
    6. 获取用户信息并登录                            │
         │ ─────────────────────────────────────> │
         │                                        │
```

**简单来说：**
1. 用户点击 "Second Me 登录"
2. 跳转到 Second Me 授权页面
3. 用户同意授权
4. Second Me 返回授权码给你的应用
5. 你的应用用授权码换取访问令牌
6. 用访问令牌获取用户信息并登录

## 第三部分：配置回调 URL

### 步骤 3.1：配置 OAuth 参数

在 "OAuth 配置" 页面，填写回调 URL：

```
┌────────────────────────────────────────────────────────────┐
│  OAuth 配置                                                │
│                                                           │
│  回调 URL (Redirect URI) *                                 │
│  ┌────────────────────────────────────────────────────┐   │
│  │  https://medroundtable.zeabur.app/api/auth/callback │   │
│  └────────────────────────────────────────────────────┘   │
│  ⚠️ 重要：必须与代码中配置的 redirect_uri 完全一致           │
│                                                           │
│  权限范围 (Scopes)                                         │
│  ─────────────────────────────────────────────────────    │
│                                                           │
│  ☑️ profile        获取用户基本信息（必需）                │
│  ☑️ email          获取用户邮箱地址（推荐）                │
│  ☑️ shades:read    读取用户的AI Shades（推荐）             │
│  ☐ softmemory:read 读取用户记忆数据（可选）                │
│                                                           │
│  ┌────────────────────────────────────────────────────┐   │
│  │              保存配置                              │   │
│  └────────────────────────────────────────────────────┘   │
│                                                           │
└────────────────────────────────────────────────────────────┘
```

### 回调 URL 详解

**回调 URL 是什么？**

用户授权后，Second Me 会将用户重定向到这个 URL，并附带授权码。

**如何确定回调 URL？**

格式：`https://你的域名/api/auth/callback`

根据你的部署方式填写：

| 部署方式 | 回调 URL 示例 |
|---------|--------------|
| Zeabur | `https://medroundtable-xxx.zeabur.app/api/auth/callback` |
| Railway | `https://medroundtable.railway.app/api/auth/callback` |
| 自定义域名 | `https://api.medroundtable.com/api/auth/callback` |
| 本地开发 | `http://localhost:8000/api/auth/callback` |

**⚠️ 重要注意事项：**

1. **协议必须一致**：如果配置的是 `https://`，就不能用 `http://`
2. **路径必须一致**：必须是 `/api/auth/callback`
3. **域名必须一致**：必须与 Zeabur 分配的域名完全一致
4. **末尾不要加斜杠**：应该是 `/callback` 而不是 `/callback/`

### 权限范围说明

| Scope | 权限说明 | 必需性 | 用途 |
|-------|---------|-------|------|
| `profile` | 获取用户ID、昵称、头像 | ✅ 必需 | 识别用户身份 |
| `email` | 获取用户邮箱地址 | ⭐ 推荐 | 用户联系、通知 |
| `shades:read` | 读取用户的AI Shades列表 | ⭐ 推荐 | 显示用户的AI助手 |
| `softmemory:read` | 读取用户的SoftMemory数据 | ⭕ 可选 | 高级功能 |

**建议勾选：** `profile` + `email` + `shades:read`

## 第六部分：常见问题

### Q1: "redirect_uri_mismatch" 错误

**现象：**
授权后显示错误：`redirect_uri_mismatch`

**原因：**
Zeabur 中的 `SECONDME_REDIRECT_URI` 与 Second Me 开发者中心配置的回调 URL 不一致。

**解决：**
1. 检查 Second Me 开发者中心中的回调 URL
2. 检查 Zeabur 中的 `SECONDME_REDIRECT_URI`
3. 确保两者完全一致（包括协议 https/http）

---

### Q2: "invalid_client" 错误

**现象：**
授权后显示错误：`invalid_client`

**原因：**
Client ID 或 Client Secret 错误。

**解决：**
1. 重新复制 Client ID 和 Secret
2. 检查是否有额外空格
3. 确认环境变量已保存并重新部署
4. 如果 Secret 泄露，在 Second Me 开发者中心点击"重新生成"

---

### Q3: 授权后没有返回用户信息

**现象：**
用户授权后，无法获取用户信息。

**原因：**
Scope 未包含 `profile`。

**解决：**
1. 在 Second Me 开发者中心
2. 进入应用 → OAuth 配置
3. 勾选 `profile` 权限
4. 保存并重新测试

---

### Q4: 本地测试正常，线上失败

**现象：**
本地开发环境 OAuth 正常，部署后失败。

**原因：**
环境变量未正确设置或域名不匹配。

**解决：**
1. 检查 Zeabur 环境变量是否正确
2. 确认 `SECONDME_REDIRECT_URI` 使用的是线上域名
3. 在 Second Me 开发者中心添加线上回调 URL
4. 重新部署服务

---

### Q5: 如何支持多个回调 URL？

**方法：**
在 Second Me 开发者中心，可以添加多个回调 URL（用逗号分隔或分行）：

```
https://medroundtable.zeabur.app/api/auth/callback,
http://localhost:8000/api/auth/callback
```
